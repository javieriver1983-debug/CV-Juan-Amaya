- Diseño moderno tipo landing page

- Animaciones suaves (scroll o hover)

- Tipografía moderna

IMPORTANTE:
- Todo debe funcionar sin frameworks ni librerías externas

- Código separado en index.html, styles.css y script.js
- Listo para subir a GitHub Pages, Netlify o Vercel
- Hazlo responsive

- Todas las rutas deben ser relativas (ej: ./assets/... )
- El proyecto debe funcionar abriendo index.html directamente sin servidor

- Todos los nombres de archivos generados deben estar en minúsculas
- No usar nombres con hash o nombres dinámicos (ej: foto-perfil-DtQeNkfJ.jpeg)
- Usar nombres simples y fijos para imágenes (ej: perfil.jpg, trabajo1.jpg, trabajo2.jpg)

- Todos los archivos deben seguir esta estructura:

/index.html
/styles.css
/script.js
/assets/
   perfil.jpg

- Todas las rutas a archivos en el HTML deben usar rutas relativas con ./ 
  (por ejemplo: ./assets/perfil.jpg, ./styles.css, ./script.js)

- No usar rutas absolutas como /assets/
- Asegurar compatibilidad con GitHub, Netlify y Vercel
- Asegurar que todas las imágenes carguen correctamente en servidor Linux (case sensitive)
